# 扩展方案 — 从笔记本演示到生产部署

## Phase 1：笔记本演示（当前）

| 组件 | 方案 | 说明 |
|------|------|------|
| 服务器 | 本地 Windows 笔记本 | WSL2 或直接运行 |
| 数据库 | SQLite | 单机无需额外服务 |
| 板子数量 | 1-5 块 | USB 串口 + 局域网 SSH |
| 用户数 | 1-2 人 | |
| AI | Anthropic API | 需要 ANTHROPIC_API_KEY |

## Phase 2：团队使用（数十人，数百板子）

### 服务器硬件

| 设备 | 规格 | 数量 | 用途 |
|------|------|------|------|
| 机架服务器 | 32核/64GB RAM/1TB SSD | 1台 | 主服务 + 数据库 |
| GPU 节点 | 24GB VRAM 显卡 | 1台（可选） | 本地部署 LLM |

### 架构升级

```
                          ┌──────────────┐
                          │   Load       │
                          │   Balancer   │
                          └──────┬───────┘
                          ┌──────┴───────┐
                 ┌────────▼────────┐ ┌───▼────┐
                 │  Web Server     │ │  AI     │
                 │  (FastAPI x N)  │ │  Node   │
                 └────────┬────────┘ └───┬────┘
                 ┌────────▼────────┐     │
                 │  PostgreSQL     │     │
                 │  + Redis        │     │
                 └────────┬────────┘     │
                 ┌────────▼────────┐ ┌───▼────────────┐
                 │  Board Proxy    │ │  RabbitMQ      │
                 │  (每板一个代理)  │ │  Task Queue    │
                 └─────────────────┘ └────────────────┘
```

### 数据库升级

| 组件 | 替换方案 | 原因 |
|------|---------|------|
| SQLite → PostgreSQL | 主数据库 | 支持并发写入、多用户 |
| + Redis | 缓存 + 会话 | 加速读取、预约锁 |

### 板子连接代理

当板子数量 > 20 时，每块板子配一个**轻量代理进程**（在板子上运行或通过树莓派做网关）：

```python
# 板子代理 (运行在板子上)
# 功能: 维持与服务器的 WebSocket/SSH 连接
#       接收指令 → 执行 → 回传结果
#       心跳上报
#       断线自动重连
```

Proxy 模式解决了：笔记本不能同时维持几百个 SSH 连接的问题。

### AI 升级选项

| 方案 | 延迟 | 成本 | 隐私 | 推荐场景 |
|------|------|------|------|---------|
| Anthropic API | 低 | 按量付费 | 数据上云 | 初期推荐 |
| 本地 vLLM + 开源模型 | 中 | 一次性硬件 | 完全本地 | 数据敏感 |
| 混合模式 | 灵活 | 灵活 | 灵活 | 生产环境 |
