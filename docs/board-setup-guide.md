## 2. 模式A: 本地 SSH (Linux板)

> 适用: Raspberry Pi / Jetson / Orange Pi / BeagleBone 等运行 Linux 的板子

### 2.1 硬件连接

```
[服务器] ──网线──→ [交换机] ──网线──→ [板子 RJ45口]
                      ↑
            如果板子有WiFi也可以走WiFi
```

**你需要:**
| 设备 | 数量 | 说明 |
|------|------|------|
| 交换机/路由器 | 1台 | 服务器和板子插同一个交换机 |
| 网线 | 板子数×1 | 六类网线, 0.5-1米 |
| USB电源 | 板子数×1 | 给板子供电 |

### 2.2 板子端配置

**Step 1: 确认板子 IP**

在板子上执行:
```bash
# 查IP地址
ip addr show
# 或
ifconfig

# 记录类似这样的地址: 192.168.1.101
```

**Step 2: 启用 SSH**

```bash
# Raspberry Pi:
sudo raspi-config → Interface Options → SSH → Enable

# 其他Linux板:
sudo systemctl enable ssh
sudo systemctl start ssh
```

**Step 3: 设置固定 IP (推荐，避免重启后IP变化)**

```bash
# 方法1: 路由器 DHCP 预留 (推荐)
# 登录路由器管理页面 → DHCP设置 → 给板子MAC地址分配固定IP

# 方法2: 板子自身设置静态IP
sudo nano /etc/netplan/01-netcfg.yaml
# 内容:
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses: [192.168.1.101/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]

sudo netplan apply
```

**Step 4: 验证连通**

从服务器(你的电脑)测试:
```bash
# Windows:
ping 192.168.1.101

# 测试SSH:
ssh pi@192.168.1.101
# 输入密码后能登录即成功
```

## 3. 模式B: 本地串口 (MCU板)

> 适用: STM32 / ESP32 / GD32 等通过 USB 转串口连接的板子

### 3.1 硬件连接

```
[服务器] ──USB线──→ [USB转串口模块] ──杜邦线──→ [MCU板]
   │                   │
   │              ┌────┴────┐
   │              │ CH340G  │ ← 最常见的USB转串口芯片
   │              │ CP2102  │
   │              │ FT232RL │
   │              └─────────┘
   │
   ├── 如果板子少 (1-5块): 直接插电脑USB口
   │
   └── 如果板子多 (10+): 
        └── PCIe USB扩展卡 → USB Hub → 多个USB转串口模块
```

**你需要:**
| 设备 | 数量 | 预估价格 |
|------|------|---------|
| USB转串口模块 (CH340G) | 板子数×1 | ~12元/个 |
| 杜邦线 (母对母) | 板子数×4 | ~3元/条 |
| USB Hub (10口带供电) | 每10块板配1台 | ~150元 |
| 多口USB充电器 | 每10块板配1台 | ~100元 |

### 3.2 物理接线

```
USB转串口模块      板子
    GND  ──────── GND     (必须接!)
    TXD  ──────── RXD     (交叉接)
    RXD  ──────── TXD     (交叉接)
    5V   ──────── VCC     (可选，看供电方式)

注意: TXD→RXD, RXD→TXD (发送对接收到)
```

### 3.3 在 Windows 上确认串口号

```powershell
# PowerShell 查看串口:
Get-WmiObject Win32_SerialPort | Select-Object Name, DeviceID

# 或打开设备管理器 → 端口 (COM和LPT)
# 插拔USB转串口，看哪个COM号出现/消失
# 记录类似: COM3, COM4, COM5...
```

### 3.4 在 Linux/Mac 上确认串口号

```bash
# 列出串口:
ls /dev/ttyUSB*
ls /dev/ttyACM*
ls /dev/cu.*

# 插拔后对比:
ls /dev/tty* > /tmp/before.txt
# 插入USB转串口
ls /dev/tty* > /tmp/after.txt
diff /tmp/before.txt /tmp/after.txt
```

### 3.5 避免串口号漂移 (Linux)

创建 udev 规则，给每个 USB 转串口固定名称:

```bash
# 获取设备序列号
udevadm info -a -n /dev/ttyUSB0 | grep serial

# 创建规则文件
sudo nano /etc/udev/rules.d/99-lab-boards.rules

# 内容:
SUBSYSTEM=="tty", ATTRS{serial}=="你的序列号", SYMLINK+="ttyLAB_STM32_01"
SUBSYSTEM=="tty", ATTRS{serial}=="另一个序列号", SYMLINK+="ttyLAB_ESP32_01"

# 生效
sudo udevadm control --reload-rules
sudo udevadm trigger
```

## 4. 模式C: 远程代理 (任意板子)

> 适用: 板子不在局域网内、跨防火墙、异地实验室、客户现场

### 4.1 工作原理

```
[板子]              [互联网]               [服务器]
  │                                           │
  │   python agent.py                          │
  │   --server ws://公网IP:8000/ws/board       │
  │   --token BOARD_TOKEN                      │
  │                                           │
  └──────── WebSocket ────────────────────────→│
            (板子主动向服务器发起连接)
            连接后服务器可向板子下发命令
```

**核心要点:** 板子主动连接服务器，所以板子不需要公网IP，也不需要配置防火墙/端口映射。

### 4.2 板子端操作

**Step 1: 安装依赖**

```bash
# 在板子上
pip install websockets
# 或
pip3 install websockets
```

**Step 2: 获取 agent.py 脚本**

```bash
# 方法1: 从GitHub下载
git clone https://github.com/cringe-sigma/Lab-Orchestrator.git
cp Lab-Orchestrator/board-agent/agent.py /opt/lab-agent/

# 方法2: 手动传文件
# 用 scp 从你的电脑传到板子:
scp board-agent/agent.py pi@192.168.1.101:/opt/lab-agent/

# 方法3: 直接从服务器下载(如果服务器HTTP可用)
# curl -o agent.py http://服务器IP:8000/static/agent.py
```

**Step 3: 启动代理**

```bash
cd /opt/lab-agent/
python agent.py --server ws://你的服务器IP:8000/ws/board --token 系统生成的TOKEN

# 看到类似输出表示成功:
# [Agent] 正在连接服务器: ws://192.168.1.100:8000/ws/board
# [Agent] 已连接到服务器
# [Agent] 注册成功: board_id=5
```

### 4.3 设置开机自启 (Linux 板)

```bash
# 复制 service 文件
sudo cp board-agent/lab-agent.service /etc/systemd/system/
# 编辑 token 和 server 地址
sudo nano /etc/systemd/system/lab-agent.service

# 修改这两行:
# ExecStart=/usr/bin/python3 /opt/lab-agent/agent.py --server ws://YOUR_SERVER_IP:8000/ws/board --token YOUR_BOARD_TOKEN

sudo systemctl daemon-reload
sudo systemctl enable lab-agent
sudo systemctl start lab-agent

# 检查状态
sudo systemctl status lab-agent
journalctl -u lab-agent -f
```

### 4.4 服务器公网暴露方式

要让远程板子能连上服务器，服务器需要能被外网访问:

| 方式 | 适用场景 | 配置难度 |
|------|---------|---------|
| **公网IP** | 服务器有独立公网IP | ⭐ 简单 |
| **内网穿透 (frp/ngrok)** | 服务器在家/校内网 | ⭐⭐ 中等 |
| **VPN (WireGuard/Tailscale)** | 安全要求高 | ⭐⭐⭐ 较难 |
| **云服务器中转** | 专业部署 | ⭐⭐ 中等 |

**推荐起步方案 (Tailscale 免费):**
```bash
# 在服务器和所有远程板子上安装 Tailscale
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

# 服务器 IP 变成 100.x.x.x
# 板子 agent 连接: ws://100.xxx.xxx.xxx:8000/ws/board
```

└─────────────────────────────────────────────────┘
```

### 6.2 预约规则

| 规则 | 说明 |
|------|------|
| 排他性 | 同一时间段一块板子只能被一人预约 |
| 时长限制 | 单次最长 4 小时 |
| 提前取消 | 未开始的预约可随时取消 |
| 超时释放 | 预约结束后自动释放板子 |

### 6.3 预约列表

```
┌──────────────────────────────────────────────────┐
│ 📅 GPIO接口实验                   [待开始]       │
│ 2026/7/25 14:00:00 — 2026/7/25 16:00:00          │
│ 板子 #1                     [取消]               │
└──────────────────────────────────────────────────┘

状态说明:
  待开始 = 预约已创建，等待时间到达
  进行中 = 当前正在使用板子
  已完成 = 预约时间结束
  已取消 = 用户主动取消
```

### 6.4 完整预约使用流程

```
1. 你创建预约 (状态: 待开始)
         │
2. 时间到达开始时间 → 系统自动激活 (状态: 进行中)
         │
3. 在你的时间段内操作板子 (执行命令/运行实验)
         │
4. 时间到达结束时间 → 自动释放板子 (状态: 已完成)
         │
5. 其他用户可以预约该板子的后续时间段
```

## 8. 用户操作完整流程示例

### 场景: 小明要用树莓派做 GPIO 实验

```
Step 1: 登录
  打开 http://lab-server:5173
  输入用户名/密码 → 登录

Step 2: 查看可用板子
  顶部 → 板子管理
  看到 Raspberry Pi 5-01 状态为 "在线"
  点击 "检查" 确认连通

Step 3: 预约
  顶部 → 预约 → 新建预约
  选择 Raspberry Pi 5-01
  时间段: 明天 14:00-16:00
  标题: GPIO 实验
  确认提交

Step 4: 到时间使用
  14:00 到达 → 预约自动激活
  去板子管理 → 找到板子 → 执行命令
  或用 AI 助手直接说 "帮我做 GPIO 测试"

Step 5: 结束后
  预约自动结束 → 板子释放
  去实验页面查看刚才的实验记录
```

### 场景: 远程协作者小李要操作北京的板子

```
Step 1: 管理员添加远程板子
  板子管理 → 添加板子 → 连接方式: 远程代理
  系统生成 Token: AbCdEf123456...
  把 Token 发给小李

Step 2: 小李在板子上启动代理
  ssh 登录板子

## 9. 常见问题排障

### SSH 连接失败

| 现象 | 检查步骤 |
|------|---------|
| ping 不通 | 板子是否开机？网线是否插好？IP 是否正确？ |
| ping 通但 SSH 连不上 | SSH 服务是否开启？`systemctl status sshd` |
| 认证失败 | 用户名/密码是否正确？ |
| 连接超时 | 防火墙是否开放 22 端口？ |

### 串口连接失败

| 现象 | 检查步骤 |
|------|---------|
| 串口不在列表中 | USB转串口驱动是否安装？ |
| 打开串口失败 | 串口是否被其他程序占用(PuTTY/串口助手)？ |
| 有输出但乱码 | 波特率是否正确？ |
| 无输出 | TX/RX 是否交叉？GND 是否连接？ |

### 远程板子掉线

| 现象 | 检查步骤 |
|------|---------|
| 状态频繁变离线 | 网络不稳定？增加 heartbeat 频率 |
| 连不上服务器 | 服务器 IP 是否变了？网络是否可访问？ |
| Token 无效 | 重新添加板子生成新 Token |
| 连接被拒绝 | 服务器端口 8000 是否开放？ |

### 预约相关

| 现象 | 检查步骤 |
|------|---------|
| 无法预约 | 检查时间是否冲突(该时段已被别人预约) |
| 预约后无法操作板子 | 预约是否已到达开始时间？状态是否为"进行中"？ |
| 板子被锁定 | 检查是否有别人正在使用 |
