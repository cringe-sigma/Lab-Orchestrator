## 板子规模方案

### 方式一：纯 Linux 板（推荐，可 Docker）

| 设备 | 单价 | 100块总价 | 特点 |
|------|------|-----------|------|
| Raspberry Pi 5 (8GB) | ~600 | ~60,000 | 生态最好 |
| Orange Pi 5 (8GB) | ~500 | ~50,000 | 性价比高 |
| 二手 Thin Client + 转接 | ~300 | ~30,000 | 最便宜 |

### 方式二：MCU + 网关代理

| 设备 | 单价 | 100块总价 | 特点 |
|------|------|-----------|------|
| ESP32-S3 | ~40 | ~4,000 | 便宜，WiFi直连 |
| STM32F4 | ~80 | ~8,000 | 工业常用 |
| 树莓派 Zero 2W (做网关) | ~150 | ~3,000(20个) | 做串口转网络 |

### 混合架构（推荐）

```
20块 Linux 板 (RPi/Orange Pi)   — 跑 Docker，做复杂实验
80块 MCU 板 (ESP32/STM32)       — 做嵌入式裸机实验
         │
         └── 每 10-20 块 MCU 配 1 个树莓派做网关代理
             代理负责: 供电 / 串口复用 / 固件烧录通道
```

## 运维建议

1. **PXE 网络启动**：批量给 Linux 板装系统，不用插 SD 卡
2. **集中供电**：USB 集线器供电柜，统一控制重启
3. **串口集线器**：USB 转 16 路串口模块，一台电脑管 16 块 MCU
4. **板子标签**：每块板子贴二维码，扫码可查在系统中的 ID 和位置
5. **分区分组**：按项目/实验类型分 VLAN 隔离
